# MobileBERT_Silksong

---

## 1. 개요
현재는 과거와 다르게 인터넷의 발달으로 게임을 실시간으로 업데이트 할 수 있다.
게임을 얼리엑세스로 출시한다음 부족한 부분을 개선하는 방식도 많이 사용하고 있고, 정식출시한 게임이라도 꾸준한 업데이트를 통해 인기를 이어가는 경우도 있다.
리뷰를 분석하고 유저들의 불만과 게임에 대한 의견을 받아들여 게임을 더 발전시킬 수 있다.

인디 게임 제작사 팀 체리(Team Cherry)의 할로우 나이트(Hollow Knight)는 메트로배니아 장르의 정점이자 기념비적인 명작으로 추앙받는다.
수려한 아트워크, 웅장한 사운드트랙, 그리고 정교하고도 어려운 난이도의 전투 시스템은 수백만 명의 팬덤을 형성했다.
그로부터 수년간의 기다림 끝에 출시된 속편 할로우 나이트 실크송(Hollow Knight: Silksong)은 게임 역사상 가장 거대한 기대와 압박을 동시에 안고 세상에 나왔다.

매우 큰 기대를 받은 만큼 실크송은 출시 전 스팀 위시리스트 등록 1위를 달성하였고,
출시일 당시 수많은 사람들이 접속하면서, 스팀 스토어 페이지에 일시적 접속 장애가 발생하였다.


현재 실크송은 스팀(Steam) 플랫폼에서 약 93%의 매우 긍정적인 평가를 기록하고 있다.
그러나 전작의 96%에 비해 부정적인 리뷰가 늘어난 것 또한 사실이다.
수년간 게임을 기다려온 콘크리트 팬층 중 일부가 돌아서서 비추천을 누른 배경에는 단순한 어려움을 넘어선 게임 디자인적 호불호 요인이 존재한다.
스팀 어워드에서 '가장 도전적인 게임(Best Game You Suck At)' 부문을 수상한 것만 봐도 알 수 있듯, 이 게임의 난이도와 시스템은 끊임없이 논쟁의 중심에 있다.

자연어 처리 모델인 MobileBERT를 활용해 수만 건의 실크송 영어 리뷰 데이터를 긍정과 부정으로 분류하여 이 게임의 장단점을 알아보고, 평가가 엇갈리게 된 원인을 분석한다.

### 2.2 탐색적 데이터 분석

직접 데이터를 읽어보았을 때 추천 여부와 리뷰 텍스트의 내용이 대체로 일치하는것으로 보였다.

<img src="/graph/text_len_count.png">

총 140335건의 리뷰 중 10자 이하의 매우 짧은 리뷰가 26149개, 10-500의 중간 리뷰가 97978개, 500자 이상의 긴 리뷰가 15874개 있다.

<img src="graph/data_compare_graph.png">

10자 이하의 리뷰와 500자 이상의 리뷰를 제거하고,
추가로 알파벳, 숫자, 공백, 문장부호 비율이 70%이하인 리뷰를 제거하여, 영어가 아니거나 이모지로 도배된 리뷰를 제거했다.
그 결과 총 96677개의 데이터가 정제되었다.

<img src="graph/silksong_reviews_playtime_graph.png">

스팀의 환불 보장 플레이타임 2시간과 대략적으로 실크송 엔딩에 도달할 때까지의 30시간을 기준으로 하여 유저를 3가지로 분류하였고,
리뷰의 추천 여부와 리뷰 작성 이후로 2시간 이상 추가로 플레이하였는지에 따라 또다시 유저의 성향을 4가지로 분류하여 그래프로 나타내보았다.


## 3. 학습 데이터 구축

<img src="graph/silksong_final_train.png">

위에서 나눈 유저 성향과 리뷰의 추천 여부를 기준으로 층화 추출을 하여 총 2000개의 데이터를 학습 데이터로 구축하였다.


### 3.1 데이터 라벨링

2000개의 학습 데이터를 직접 확인하고 부정, 긍정, 중립, 관련없음의 4가지 요소로 라벨링을 진행했다.

- 라벨링 기준
  - 0(부정): 부정적인 내용만 있을 경우.
  - 1(긍정): 긍정적인 내용만 있을 경우.
  - 2(중립): 감정 없는 사실만 서술하거나. 긍정/부정 의견이 둘 다 있는 경우.
  - 3(관련없음): 같은 단어 도배, 게임에 관련된 이야기가 아닌것, 영어가 아닌 리뷰

다른 내용 없이 그냥 "쉽다", "어렵다"라고 서술된 것은 중립으로 분류하였고, "어렵지만 재미있다."나 "너무 어려워서 플레이할 수가 없다." 등 감정이 드러나는 것은 긍정/부정으로 분류하였다.\
비추천 리뷰 중 어떤 보스나 지역 이름만 적는 경우가 꽤 있었는데 이것은 해당 보스나 지역에 대한 불만으로 판단하여 부정으로 분류하였다.

## 5. 문제제기에 대한 결과
부정 키워드를 통해 부정적 평가의 주요 원인을 알아본다.\
불합리한 난이도 디자인 (예: 2 masks, damage, punishment) — 전작보다 가혹해진 페널티에 대한 불만.
지루해진 동선과 피로감 (예: runback, tedious, bench) — 보스방이나 다음 구역까지 가는 과정이 너무 멀고 지친다는 비판.


## 6. 느낀점 및 개선방향
게임리뷰의 특성상 인터넷 유행어나, 인게임 고유명사, 오타 등의 요소가 많이 포함되어있어 데이터 라벨링이 까다롭다고 느꼈다.
게임 플레이타임, 리뷰 작성 시간 등 수집한 데이터를 전부 활용하지 못한 것이 아쉽다.
문제에 대한 결과 분석이 빈약하다. 더 보기 쉽도록 시각화할 필요가 있어 보인다.
